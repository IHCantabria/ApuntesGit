Características y comandos¶

Git trabaja como un sistema distribuido, podemos trabajar contra nuestro repositorio sin conectarnos remotamente a GitHub.

clone¶

El método clone descarga un repositorio en nuestro equipo. Contiene todo el histórico y referencias hacia el repositorio remoto.

init¶

Iniciamos un nuevo repositorio local. Genera una carpeta oculta llamada .git, que contiene toda la información necesaria para trabajar.

add & commit¶

Cuando queremos grabar los cambios en git, debemos especificar cuáles son los ficheros que nos interesan, con add los pasaremos a estado staged, posteriormente con commit confirmamos que queremos grabar esos ficheros, y especificaremos un mensaje.

pull / push¶

Para traernos los cambios del repositorio remoto, utilizamos el comando pull. Descargará la rama especificada y hara un merge de los ficheros, las historias de local y remoto quedarán igualadas. Cuando queremos enviar las novedades de nuestro repositorio al remoto, utilizaremos el comando push.

fetch¶

Para traernos los cambios del repositorio remoto, pero a diferencia de pull, en este caso no hará el merge de los ficheros, que será un paso que tengamos que realizar nosotros.

branch¶

Crea una nueva rama a partir de la actual.

checkout¶

Cambia la rama actual a la que decidamos o incluso a un commit.

merge¶

Trae los cambios de una rama a la actual, combinando los cambios.

tag¶

Etiqueta un commit con un nombre que decidamos.

log¶

Muestra el listado de commits y su información: autor, fecha y mensaje.
